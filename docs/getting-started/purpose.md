# 入门

在我们正式开始之前，我们需要先问自己几个一个问题。

## 我是否具备基本的计算机理论知识？

此处的“知识”主要是指诸如分区，引导以及操作系统相关知识等等基本的操作系统运维概念，不需要深入了解。

如果你在实践的过程中发现了不太了解的名词，可以看一看我们编写的“概念”部分来补课。

!!! warning "警告"
      我们不推荐小白跳过理解直接安装，**否则可能导致您所使用设备的操作系统产生不稳定性或发生严重损坏！！！造成的损失不由LinuxCabin非营利组织以及文档贡献者承担！！！**


## 我为什么要使用Linux？

尽管看起来可能有点啰唆，但这是一个非常重要的问题，它从会根本上决定我们后续的所有选择。

### 1. Windows太烂了，想要叛逃Linux

是的，Microslop越来越多的垃圾广告、卡顿和充满bug的系统、爆满的C盘等等等等的问题，的确令用户难以忍受。所以，我们选择了Linux，它简洁，轻量，没有烦人的商业化行为，的的确确是很好的选择。

不过，下面有一些需要您在这之前检查的事情：

1. 我日常所需的软件是否有Linux版本/替代品？  
   如果没有，您可以参考“兼容性”这一章节(WIP)。
2. 我能否忍受可能会带来的不便与学习成本？  
   这包括可能会和终端打一部分交道，以及可能存在的不稳定性风险。
3. 我能否完全脱离Windows？  
   如果不能，您需要参考“虚拟机”、“兼容层”等章节(WIP)。

检查完毕！打开下一章，开始我们的旅途吧！

### 2. 我只是单纯好奇，想要尝试/把玩下

你好，开拓者！Linux的广大社区也是像你这样千千万万的具有开拓精神的爱好者用爱发电而形成的，希望你旅途愉快！

不过请注意，我们的教程主要偏向的是“用”这一部分，如果是想要诸如极客那样的体验，还请移步别处啦 ;-D。

相信有这样的精神，你一定能够克服接下来的重重阻碍，打开下一章，let's go 🚀！

### 3. 我向往自由（软件）

欢迎来到自由世界！Linux自创立之初就是自由软件（准确来说是创立之后不久），并且极大地推动了世界自由软件的发展，是兼顾“支持”与“自由”的选择。

我们的教程在编写的时候也会给各个发行版以及相应的软件包标上是否自由的标签，您可以放心阅读。

点击下一章，开始你的自由旅程吧。

### 4. 我喜欢Linux酷炫的视觉效果

我们很理解，作为一名小白，在看到网上那些炫酷的桌面的时候，你内心是怎样的震撼。什么具有科技感的状态栏，炫酷的图标，都比Windows要炫酷不少。

不过，如果单纯是因为视觉效果而入坑，您一定要做好以下的心理准备：

1. 可能与终端打一点小交道
2. 你的配置效果很有可能让你感到不尽人意
3. 我们的教程并不全面，只是为入坑而写，如果需要更深入的知识推荐去[Arch Wiki](https://wiki.archlinux.org/title/Main_page) | [Arch Wiki 中文版](https://wiki.archlinuxcn.org/wiki/%E9%A6%96%E9%A1%B5)深造

并且，您一定要明辨下面的几个概念：

 - 桌面环境 (DE)
 - 发行版
 - 窗口管理器 (WM)
 - 更新模式

好啦，既然你已经准备好了，那我们就向下继续吧！

### 5. 我就是想装逼

如果是这样，我们并不会推荐您使用Linux。众所周知，目前Linux社区“恃才傲物”的问题有很大一部分都是“嘉豪”一类人所造成的。所以，如果您只是单纯想要炫耀自己的技术力，我们不推荐通过Linux而达成这个目的，并且秩序与风气良好的社区是极力反对这种现象的。

当然，如果您依然想要继续，那么则希望您能与其他用户平等交流，少一些炫耀，多一些真才实干，为社区创造一个相对好的氛围。
